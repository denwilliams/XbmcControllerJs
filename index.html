XbmcControllerJs

JavaScript library that makes it easy to talk to XBMC via the JSON API using Websockets, with a fallback to the HTTP API. Note: HTTP API currently requires the JSON RPC 2.0 jQuery plugin (https://github.com/datagraph/jquery-jsonrpc), and subsequently jQuery. Later versions will be fully self contained.

Components

XbmcControllerJs consists of 3 main components/classes:

  • XbmcWebSocketsApi
  • XbmcHttpApi
  • XbmcController

XbmcController uses the other two classes. Typically you will only need to use XbmcController, but it is possible to communicate with XbmcWebSocketsApi or XbmcHttpApi directly.

XbmcController

Usage:

var xbmc = new XbmcController(); // automatically determines websockets or http
xbmc.Application.SetMute({mute:true});

XbmcWebSocketsApi & XbmcHttpApi

Both classes follow the same interface. The only difference between the two is the notifications that can be subscribed to.

Usage:

var xbmcWs = new XbmcWebSockets('192.168.0.10'); // <-- XBMC IP or Hostname
xbmc.subscribe('Application.OnVolumeChanged',onVolChanges);
xbmcWs.call('Application.SetMute',{mute:true}); // <-- will cause onVolChanged to be called
xbmcWs.unsubscribe('Application.OnVolumeChanged',onVolChanges);

function onVolChanged(response) {
    console.log('New volume is '+reponse.volume);
    console.log('New mute is '+response.mute);
}
